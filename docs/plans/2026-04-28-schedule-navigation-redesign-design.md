# 课表导航与界面重设计文档

日期：2026-04-28  
状态：已确认  
范围：Android App 入口壳层、课表页、插件页、设置页

## 1. 目标

- 底部导航改为 `课表 / 插件 / 设置` 三项
- 底栏视觉参考 GKD 截图，采用深色悬浮容器与选中胶囊态
- 课表主页参考第二张截图，改为周视图排课布局
- 保留现有课表同步、插件加载、提醒创建等业务能力，不重写数据层

## 2. 设计边界

- 仅重构 Compose UI 与应用壳层，不调整插件工作流、仓储协议与提醒存储模型
- 不引入新的业务模块，设置页先作为宿主页面承载应用信息与状态卡片
- 课表仍基于现有 `TermSchedule` 与 `TermTimingProfile` 渲染

## 3. 页面结构

### 3.1 应用壳层

- `MainActivity` 使用单一 `Scaffold`
- 内容区根据当前选中的底栏项切换 `课表 / 插件 / 设置`
- 底栏固定为三项，图标居中，文字置底，选中项使用圆角胶囊背景

### 3.2 课表页

- 顶部显示当前日期、周次与星期信息
- 增加周偏移切换，允许浏览前后周
- 主区域为周视图网格：
  - 列：周一到周日
  - 行：节次时间段
  - 单元中的课程块按 `dayOfWeek/startNode/endNode` 绝对定位
- 课程块点击后显示详情卡片，并保留提醒创建入口
- 原有账号、密码、学期、插件切换与同步按钮收纳为“同步设置”区域，避免破坏课表主体

### 3.3 插件页

- 保留远程市场、本地导入、安装预检、移除插件等功能
- 统一成深色卡片与更大圆角的视觉语言

### 3.4 设置页

- 新增风格统一的宿主设置页
- 首版承载应用说明、当前状态、提醒能力说明与版本信息
- 不承诺复杂设置持久化，仅提供可进入、可浏览、视觉完整的页面

## 4. 数据与交互

- 周视图直接读取 `ScheduleUiState.schedule` 与 `timingProfile`
- 当 `timingProfile` 缺失时，回退为根据课程节次推导最小可用时间轴
- 周次基于 `termStartDate` 和当前/目标周计算
- 提醒规则仍调用现有 `onSelectCourse / onSelectTimeSlot / onCreateReminder`

## 5. 风格方向

- 主色调：深蓝黑背景、低饱和紫蓝选中态、彩色课程块
- 卡片：大圆角、轻层次、低对比描边
- 版式：更多留白，弱化传统表单堆叠感，突出课表网格主体

## 6. 验证方式

- `assembleDebug` 编译通过
- 手动检查三项底栏切换是否正常
- 手动检查课表在有数据、无数据、无节次配置时的展示是否可用
- 手动检查插件页现有安装/移除入口是否仍可操作

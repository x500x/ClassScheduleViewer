# 长江大学 EAMS 插件与课表页收敛设计

日期：2026-04-30

范围：课表页、设置页、V2 工作流插件引擎、内置长江大学插件

## 1. 目标

- 课表页只保留课表主体、周切换和必要状态提示。
- 宿主不再暴露账号、密码、学期、基础地址表单；同步入口移动到当前插件卡片。
- 新增长江大学教务系统插件，支持通过受限 Web 登录后抓取课表。
- 在不回退到旧 QuickJS 插件模型的前提下，为 V2 工作流引擎补充可复用的 EAMS 课表解析能力。

## 2. UI 方案

### 2.1 课表页

- 保留：顶部周信息、已安装插件切换、周课表网格、插件横幅、同步状态提示。
- 移除：账号密码表单、基础 URL、学期 ID 输入、同步按钮、提醒规则列表、提醒创建表单。
- 交互调整：
  - 课表块点击后不再在课表页展开复杂配置。
  - 无插件时继续提示去插件页安装。
  - 无课表时提示去插件页的当前插件卡片完成同步。

### 2.2 设置页

- 承接提醒相关配置与状态说明：
  - 当前插件与同步入口说明
  - 插件状态与提醒建议
  - 提醒规则列表与删除操作
- 若已选中课程，再展示提醒创建卡片。

### 2.3 插件页

- 仅保留长江大学插件。
- 当前插件卡片提供 `同步课表` 入口。
- 点击同步后直接在插件页内进入统一认证 Web 登录与取数流程。

## 3. 插件与引擎方案

### 3.1 约束

当前 V2 引擎只有 `web_session`、`http_request`、`schedule_emit_static`，无法把长江大学 EAMS 返回的 HTML/脚本直接转成 `TermSchedule`。

### 3.2 推荐实现

新增一个宿主内建的结构化步骤，用于解析 EAMS 课表页面：

- 步骤从工作流上下文读取：
  - Web 会话回传的 Cookie
  - 课表主页 GET 返回内容
  - 课表明细 POST 返回内容
- 步骤负责：
  - 解析学期与项目参数
  - 解析可用周数
  - 解析课程表脚本中的 `TaskActivity(...)` 与索引位置
  - 生成标准 `TermSchedule`
- 这样插件仍只声明流程，学校适配差异收敛到插件输入和宿主解析器参数。

## 4. 长江大学插件方案

- 作为内置 `plugin-dev` 插件随应用安装。
- 应用启动后会主动清理 demo 插件，仅保留长江大学插件。
- 登录入口使用你给出的 CAS 地址。
- 允许访问：
  - `cas-yangtzeu-edu-cn.atrust.yangtzeu.edu.cn`
  - `atrust.yangtzeu.edu.cn`
  - `jwc3-yangtzeu-edu-cn-s.atrust.yangtzeu.edu.cn`
- 工作流大致步骤：
  - `ui_emit`
  - `message`
  - `web_session`
  - `http_request` 获取课表主页与周选择信息
  - `http_request` 拉取课表明细
  - `schedule_emit_eams` 解析标准课表

## 5. 风险与处理

- EAMS 返回内容是 HTML + 内嵌脚本，解析容错要足够强，不能只依赖单个正则。
- 课表节次长度不直接在示例参数中明示，需要根据写入索引推断连续节次。
- 登录 Cookie 需要从 WebView 回传后注入 HTTP 请求头，否则后续接口无法复用登录态。

## 6. 验证

- 编译通过。
- 长江大学插件在插件页可见并可安装/选择。
- 课表页不再出现同步表单和提醒规则编辑。
- 设置页可以完成同步与提醒管理。
- EAMS 解析器能从示例结构生成标准课表。
